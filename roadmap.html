Critical path

Every phase must leave behind a testable system. The proxy comes first, the tunnel provider second, and the TUI only after the underlying event model is trustworthy.

00Foundation

Establish the project

Complete

Create a modern Python package with only the tooling needed to develop safely.

  • Source package and CLI entry point
  • Ruff, pytest, and type checking
  • README and example configuration
  • CI for linting and tests
Acceptance$ tunnitup --help
01Routing core

Move the first request

Complete

Build the reverse proxy independently of any public tunnel provider.

  • Longest-prefix route matching
  • Methods, queries, headers, and bodies
  • Optional path-prefix stripping
  • Local-only binding and useful 502 errors
AcceptanceGET / → :3000
POST /api/users → :8000/users
02Correctness

Make the proxy dependable

Complete

Turn the happy-path prototype into infrastructure that behaves predictably under failure.

  • Streaming bodies and large uploads
  • Forwarded and hop-by-hop headers
  • Connection and response timeouts
  • Disconnects and graceful shutdown
  • Route conflict validation
  • Failure-focused integration tests
03Configuration

Make it pleasant to operate

Complete

Add tunnitup.toml, validation, project initialization, and readable startup output.

  • tunnitup init
  • tunnitup validate
  • Clear route summaries
  • Actionable configuration errors
Acceptance$ tunnitup proxy
04ngrok

Connect the public tunnel

v0.1 target

Supervise an installed ngrok CLI and expose the proxy through the user's permanent domain.

  • CLI detection and auth checks
  • Proxy-port tunnel startup
  • Public URL discovery
  • Unexpected-exit reporting
  • Coordinated shutdown
  • Provider diagnostics
Acceptance$ tunnitup up --provider ngrok
One command starts the proxy, tunnel, and every configured route.
First releasable version · v0.1
05Observability

Show what the system is doing

Operations

Expose useful signals before designing the terminal interface that will consume them.

  • Method, path, route, and status
  • Duration and active requests
  • Upstream health checks
  • Secret-aware log redaction
  • Human and structured output
  • Quiet and verbose modes
06Advanced HTTP

Handle long-lived traffic

Protocols

Treat upgrades and streaming as their own correctness milestone.

  • WebSocket upgrades
  • Bidirectional forwarding
  • Server-sent events
  • Backpressure and cancellation
  • Configurable timeouts
  • Request-size limits
07Terminal UI

Make operations visible

Textual

Build a TUI that observes and controls the core without owning proxy or provider logic.

  • Route and health overview
  • Live request stream
  • Filterable logs
  • Public URL display and copy
  • Keyboard-driven control
  • Clean terminal restoration
08Providers

Move beyond one tunnel vendor

Ecosystem

Extract the provider contract from proven ngrok behavior, then widen support.

  • OutRay adapter
  • Cloudflare Tunnel adapter
  • Custom executable provider
  • Provider-specific auth guidance
  • Optional embedded ngrok SDK
  • Capability-aware configuration
09Release

Become a durable open-source tool

Community

Make installation, contribution, and maintenance predictable across platforms.

  • Cross-platform test matrix
  • PyPI and standalone builds
  • Architecture documentation
  • Security and contribution guides
  • End-to-end example project
  • Versioned releases and changelog
